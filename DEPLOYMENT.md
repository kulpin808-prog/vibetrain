# 🚀 Развертывание Fitness Trainer AI Bot

## 📋 Текущая ситуация

Сейчас бот работает **ТОЛЬКО** на вашем компьютере в режиме разработки (polling). Когда вы выключаете компьютер - бот перестает работать.

## 🎯 Цель развертывания

Бот должен работать **24/7** на сервере, чтобы пользователи могли обращаться к нему в любое время.

## 🆓 БЕСПЛАТНЫЕ варианты развертывания

### 1. **Railway** (Рекомендую - самый простой)

**Плюсы:**
- Бесплатный план (512MB RAM, 1GB storage)
- Автоматическое развертывание из GitHub
- PostgreSQL база данных включена
- Простая настройка

**Минусы:**
- Ограничения по ресурсам (может не хватить для большого количества пользователей)

**Инструкция:**
1. Зарегистрируйтесь на [railway.app](https://railway.app)
2. Подключите GitHub аккаунт
3. Создайте новый проект из репозитория
4. Добавьте переменные окружения (см. ниже)
5. Разверните проект

### 2. **Render** (Бесплатный с ограничениями)

**Плюсы:**
- Бесплатный tier (750 часов/месяц)
- Простая настройка
- Поддержка Node.js

**Минусы:**
- Спит после 15 минут без активности
- Ограниченное время работы

### 3. **Vercel** (Бесплатный)

**Плюсы:**
- Бесплатный план
- Быстрое развертывание
- Хорошая документация

**Минусы:**
- Serverless функции (не подходит для постоянного бота)
- Таймауты на запросы

## 💰 ПЛАТНЫЕ варианты ($5-20/месяц)

### **DigitalOcean App Platform** ($5/месяц)
- Стабильная работа
- Легко масштабировать
- PostgreSQL база данных

### **Heroku** ($7/месяц)
- Простая настройка

- Хорошая документация

- Полный контроль
- DigitalOcean, Linode, или Hetzner
- Нужно настраивать самому

## ⚙️ Настройка переменных окружения

Создайте `.env` файл на сервере:

```bash
# Production settings
NODE_ENV=production
PORT=3000
WEBHOOK_URL=https://your-app-domain.com

TELEGRAM_BOT_TOKEN=из_BotFather
OPENAI_API_KEY=из_OpenAI
# Тот же публичный URL, что выдал хостинг (HTTPS), без слэша в конце
TELEGRAM_WEBAPP_URL=https://ваш-сервис.up.railway.app
WEBAPP_URL=https://ваш-сервис.up.railway.app

# Add these when ready
GOOGLE_SHEETS_API_KEY=your_key_here
GOOGLE_SHEETS_SPREADSHEET_ID=your_id_here
NOTION_API_KEY=your_key_here
NOTION_DATABASE_ID=your_id_here
```

## 🛠️ Быстрое развертывание на Railway

### Шаг 1: Подготовьте код
```bash
# Добавьте эти файлы в git
git add .
git commit -m "Add production deployment files"
git push origin main
```

### Шаг 2: Создайте Railway аккаунт
1. Перейдите на [railway.app](https://railway.app)
2. Зарегистрируйтесь (GitHub аккаунт)
3. Подтвердите email

### Шаг 3: Разверните проект
1. Нажмите "New Project"
2. Выберите "Deploy from GitHub repo"
3. Выберите ваш репозиторий
4. Railway автоматически определит Node.js проект

### Шаг 3.1: Команда запуска
В настройках сервиса укажите **Start Command**:

`npm run start:full`

Так на одном порту поднимается и **Web App** (`server.js`), и **бот** (`index.js`, long polling).

### Шаг 4: Настройте переменные
В Railway dashboard:
1. Перейдите в "Variables"
2. Добавьте все переменные из `.env`
3. Добавьте `WEBHOOK_URL` (Railway предоставит домен)

### Шаг 5: Настройте webhook в Telegram
```bash
# После развертывания получите URL от Railway
# И настройте webhook:
curl -F "url=https://your-railway-app.com" https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook
```

## 📊 Мониторинг и поддержка

### Railway бесплатный план:
- **512 MB RAM** - хватит для 50-100 активных пользователей
- **1 GB Storage** - достаточно для базы данных
- **Автоматическое масштабирование**

### Когда переходить на платный хостинг:
- >100 активных пользователей в день
- Нужна кастомная база данных
- Требуется высокая надежность

## 🐛 Устранение проблем

### Бот не отвечает:
1. Проверьте логи в Railway dashboard
2. Убедитесь, что переменные окружения правильные
3. Проверьте webhook URL

### Ошибка памяти:
- Перейдите на платный план Railway
- Или оптимизируйте код (кеширование, etc.)

### Таймауты:
- Увеличьте таймауты в настройках
- Оптимизируйте AI запросы

## 📞 Поддержка

Если возникнут проблемы:
1. Проверьте логи приложения
2. Посмотрите документацию Railway
3. Напишите в чат поддержки Railway

4. **Масштабирование:** Мониторьте использование ресурсов
